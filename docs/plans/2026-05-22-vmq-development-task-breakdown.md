# Vmq 重构开发任务清单

> **说明**：本文档用于把现有需求文档、接口兼容文档、数据库设计文档和执行计划文档，进一步拆解为可直接实施的开发任务清单。本文档不包含代码实现，只定义执行顺序、任务边界、交付物和验收标准。

## 1. 是否可以进入开发实施

可以。

当前已经具备进入开发阶段的基础文档：

- 需求范围已明确
- 接口兼容协议已明确
- 数据库结构已明确
- 实施阶段顺序已明确

这意味着后续可以直接按任务包进入开发，不需要再回头补“系统到底要做什么”这一层。

不过有一个前提：

- 开发时必须严格以现有文档为准，尤其是金额占位、订单状态、监控端兼容、回调判定这些高风险逻辑

## 2. 开发实施建议

建议采用“先后端核心链路，后前端界面，最后联调回归”的顺序。

推荐开发顺序：

1. 工程骨架
2. 数据库与迁移
3. 共享规则
4. 后端核心模块
5. 前端后台
6. 支付页
7. 联调测试
8. 迁移与上线准备

## 8. 开发前最后提醒

真正开始编码前，开发团队必须统一三条红线：

1. 不要改签名规则
2. 不要改订单状态语义
3. 不要改金额占位算法结果

这三个点一旦改了，就不是“重构”，而是“重写业务规则”了。
